### 渲染问题解决（重要经验）
- **OSMesa 在此无 GPU 环境渲染为纯灰度**，不可用于可视化
- **EGL 渲染正常有颜色**，必须用 `MUJOCO_GL=egl`
- **原始 STL mesh 坐标错误！** FreeCAD 导出的顶点是米级(-34m~+38m)，不是 mm。导致机器人几何体飞到几十米外
- **修复方案**: 完全重写 electronbot.xml，用 box/cylinder/capsule 替代坏 mesh
- 模型是 mm 单位，机器人基座 z=15mm, 头部 z≈110mm, 高~95mm
- 头灯光照需要很弱：`ambient=[0.05,0.05,0.06] diffuse=[0.15,0.15,0.18]` (mm模型易过曝)
- **最终摄像头参数**: `lookat=[0,0,0.055] distance=0.04 elevation=-35` (极近距离俯视)
- scene_tabletop.xml 桌面需降到 z=0 (原 z=45.5 会让机器人陷在桌下)
- named camera(head_cam)定向有问题，改用 MjvCamera 自由摄像头

### 渲染 Bug 修复（重要！推翻旧假设）
**问题**: 用户报告渲染显示一个灰色大立方体占满画面（gray=99%, uniq=246）  
**之前的结论是错的**: "极近距离俯视"、"头灯需极弱"这些都是 znear bug 的 workaround，不是正确的配置

**根因分析**（通过自动化脚本 diagnoise_render_v3.py）:
- `scene_tabletop.xml` 中 `<map znear="0.0001" zfar="0.5"/>` 的 znear=0.0001m (0.1mm) 导致浮点精度崩溃
- 相机距离 > 5cm 时，投影矩阵异常，全部渲染为纯黑 (gray=100%, uniq=1)
- headlight 被 XML 设为全零 ambient="0 0 0" diffuse="0 0 0"，光照完全依赖场景 3 盏灯

**修复**:
1. `scene_tabletop.xml`: znear=0.001, zfar=100.0, headlight 改为非零值
2. `env.py`: cam.distance=0.25 (25cm), azimuth=145, elevation=-25, lookat=[0,0,0.04]

**修复后指标**: gray=11%, std=57, dark=9%, uniq=2823 ← 正常！
