本书特色

每节末尾均包含 「设计文档 vs 实际实现」差异。这些差异——而不是漂亮的架构图——才是本书最有价值的贡献。

如何阅读

  • 按顺序读：每节承接上一节的产物，全序列导航保证你不会迷路；
  • 动手做：所有命令、脚本、参数均来自真实环境，可直接复现；
  • 看差异：遇到「设计 vs 实现」表，先想自己会怎么选，再对照作者的取舍。
