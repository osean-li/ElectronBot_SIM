# 第一章 绪论

## 1.1 项目背景

本项目源于一个朴素的出发点：**没有真机，能不能学机器人？**

ElectronBot 是稚晖君于 2023 年开源的一款桌面机器人，由 6 个舵机驱动，全 3D 打印结构，外观精致。但它的门槛不低——你得有 3D 打印机，得买舵机和 ESP32 主控，得焊接电路，还得刷固件。对于想入门机器人开发但手头没有硬件的学习者来说，这些步骤本身就是一道屏障。

即使有了真机，AI 开发也不方便在上面直接进行——强化学习训练需要数百万次交互，真机舵机几分钟就烧了。碰撞、摔落、过度旋转——每个错误在真机上都可能是物理损坏。

为了解决这两个问题（没有硬件也能学、有硬件也不敢直接跑），本项目 ElectronBot_SIM 以 ElectronBot 机械结构为参考，以 xiaozhi-esp32 固件为真机基准，在 MuJoCo 仿真环境中完整复现了 ElectronBot 的物理模型、运动控制和传感器系统。学习者可以在没有硬件的情况下：

- 了解机器人从 CAD 图纸到仿真模型的完整转换流程
- 学习 MuJoCo 物理引擎的使用和 Gymnasium RL 环境封装
- 实践从模仿学习到强化学习的 AI 训练管线
- 掌握 MCP 协议和 Sim2Real 的部署方法

项目的核心设计目标为：**仿真中开发的 AI 策略，通过 MCP（Model Context Protocol）协议零修改部署到真机 ESP32 上。** 这意味着如果未来你获得了真机，仿真里训练好的策略可以直接运行——不需要重写代码。

整个项目的源代码、文档和工具链完全开源，主体工作在一台 RTX 2060 的普通台式机上完成，不存在硬件或算力门槛。

## 1.2 系统架构概览

平台采用 7 层分层架构，自底向上分别为：

- **Layer 1：模型描述层**。从 STEP 原始 CAD 图纸出发，按运动组合并导出 STL，生成 MuJoCo 可加载的 MJCF 格式物理模型。
- **Layer 2：物理仿真引擎**。基于 MuJoCo 3.x，提供刚体动力学、接触碰撞、摄像头渲染及域随机化能力。
- **Layer 3：动作系统层**。实现 6 个舵机的控制，提供预设动作与舵机级控制接口，并确保与真机固件行为 1:1 对齐。
- **Layer 4：MCP 协议层**。定义 AI 与机器人之间的 JSON-RPC 通信协议，仿真端（McpSimBridge）与真机端（McpCloudBridge）共享同一接口。
- **Layer 5：传感器与观测层**。提供 JointSensor、CameraSensor、ContactSensor 等传感器模块，并支持 full 与 realistic 两种观测模式。
- **Layer 6：智能决策层**。包含模仿学习（BC/ACT）、强化学习（PPO/SAC）及文本 VLA（LLM → MCP）三条训练管线。
- **Layer 7：应用与评估层**。提供 Web UI 可视化、CLI 工具、Benchmark 套件及 Sim2Real 部署桥接器。

## 1.3 本书结构

本书按项目实施顺序组织，共六章：

- **第二章**：环境搭建与首次模型加载
- **第三章**：从 STEP 图纸到 MJCF 模型
- **第四章**：Gymnasium 环境与动作系统的实现
- **第五章**：MCP 协议层的实现
- **第六章**：AI 训练管线的实现
- **第七章**：Sim2Real 部署实践

每章均以概要设计文档的对应章节为起点，按实施顺序展开，并在末尾记录实际实现与设计文档的差异。
