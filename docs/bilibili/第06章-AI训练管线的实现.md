## 6.2 传感器模块

### 6.2.1 JointSensor

```python
class JointSensor:
    def __init__(self, model, data):
        self.model = model
        self.data = data
    
    def get_pos(self):
        """返回 6 维关节角度数组（弧度）"""
        return self.data.qpos[:6].copy()
    
    def get_vel(self):
        """返回 6 维关节速度数组"""
        return self.data.qvel[:6].copy()
```

### 6.2.2 CameraSensor

```python
class CameraSensor:
    def __init__(self, model, renderer):
        self.model = model
        self.renderer = renderer
    
    def render_rgb(self):
        """返回 RGB 图像 (H, W, 3)，仿真专属"""
        mujoco.mj_forward(self.model, self.data)
        self.renderer.update_scene(self.data, camera="front")
        return self.renderer.render()
```

### 6.2.3 双观测模式

环境实现了 `_get_obs()` 方法的双模式分支：

```python
def _get_obs(self):
    if self.obs_mode == "full":
        return {
            "joint_pos": self.joint_sensor.get_pos(),
            "joint_vel": self.joint_sensor.get_vel(),
            "image": self.camera_sensor.render_rgb(),
        }
    elif self.obs_mode == "realistic":
        return {
            "commanded_joint_pos": self.data.ctrl[:6].copy(),
            "is_moving": np.array([1.0 if self._is_moving else 0.0]),
        }
```

`full` 模式包含仿真可获取的全部观测信息，用于研究和调试。`realistic` 模式仅包含真机可获取的数据，用于 Sim2Real 训练和验证。

## 6.3 模仿学习——Behavior Cloning

### 6.3.1 数据采集

```bash
python -m src.electronbot_ai.il.collect_demos \
    --task wave \
    --output demos/wave.hdf5 \
    --num_demos 50
```

每条演示记录在 HDF5 文件中，包含观测序列（joint_pos、可选图像）与动作序列（6 维角度增量）。数据格式兼容 robomimic 标准。

### 6.3.2 BC 策略训练

策略网络为一个 2 层 MLP：

```python
class BCPolicy(nn.Module):
    def __init__(self, obs_dim=6, action_dim=6):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(obs_dim, 128), nn.ReLU(),
            nn.Linear(128, 128), nn.ReLU(),
            nn.Linear(128, action_dim), nn.Tanh(),
        )
```

训练采用 MSE 损失 + Adam 优化器：

```python
dataset = HDF5Dataset("demos/wave.hdf5")
loader = DataLoader(dataset, batch_size=64, shuffle=True)
policy = BCPolicy()
optimizer = torch.optim.Adam(policy.parameters(), lr=1e-3)

for epoch in range(200):
    for obs, action in loader:
        pred = policy(obs)
        loss = F.mse_loss(pred, action)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

200 epoch 训练在 RTX 2060 上约 3-5 分钟收敛。训练后的策略可控制机器人完成基本挥手动作。

### 6.3.3 BC 的局限

1. **分布偏移（Distribution Shift）**：策略执行中的微小偏差会随时间累积，导致状态分布偏离训练数据分布。
2. **无法超越示范者**：BC 本质是复制，策略性能受限于演示数据质量。
3. **多模态任务表现差**：如抓取任务有多种可行方式，BC 会在各方式间"平均"，产生模糊动作。

BC 适合原型验证（半天出结果），但不适合复杂任务。

## 6.4 强化学习——PPO

### 6.4.1 奖励函数设计

以 `reach` 任务为例，奖励函数包含密集奖励与稀疏奖励两部分：

```python
class ReachTask(TaskBase):
    def compute_reward(self, env):
        ee_pos = env.data.body("left_hand").xpos
        distance = np.linalg.norm(ee_pos - self.target_pos)
        
        reward = -distance * 10.0        # 密集奖励：越近越好
        
        if distance < 0.01:              # 到达阈值 10mm
            reward += 10.0               # 稀疏奖励：到达大幅加分
        
        return reward
```

### 6.4.2 PPO 训练

```bash
python -m src.electronbot_ai.rl.train_ppo \
    --task reach \
    --total-timesteps 2000000 \
    --num-envs 16
```

训练过程观察（TensorBoard）：

| 步数 | 现象 |
|:---:|------|
| 0 | 奖励 ≈ 0，策略完全随机抖动 |
| 300k | 奖励明显上升，策略找到"接近目标"的方向 |
| 700k | 奖励收敛，成功率 > 80% |
| 2M | 最终成功率 ≈ 95% |

### 6.4.3 并行环境加速

PPO 训练需要大量交互数据。采用 `SubprocVecEnv` 并行 16 个 MuJoCo 环境：

```python
from stable_baselines3.common.vec_env import SubprocVecEnv

def make_env(rank):
    def _init():
        env = ElectronBotEnv(render_mode="rgb_array")
        env.seed(rank)
        return env
    return _init

env = SubprocVecEnv([make_env(i) for i in range(16)])
```

RTX 2060 12GB 可运行 16-32 个并行环境。训练速度相比单环境提升约 16 倍。

## 6.5 域随机化

为缩小仿真与真机的差距，在每次 `env.reset()` 时随机化物理参数：

```python
def apply_domain_randomization(self):
    import random
    
    for i in range(self.model.njnt):
        self.model.damping[i] *= random.uniform(0.8, 1.2)      # 阻尼 ±20%
        self.model.frictionloss[i] *= random.uniform(0.5, 1.5) # 摩擦 ±50%
    
    for i in range(self.model.nbody):
        self.model.body_mass[i] *= random.uniform(0.85, 1.15)  # 质量 ±15%
    
    for i in range(self.model.nu):
        self.model.actuator_gain[i] *= random.uniform(0.9, 1.1) # 增益 ±10%
```

随机化参数范围：

| 参数 | 范围 | 模拟的物理差异 |
|------|:---:|---------------|
| 关节摩擦 | ±50% | 3D 打印公差 + 润滑差异 |
| 关节阻尼 | ±20% | 舵机齿轮阻尼 |
| 执行器增益 | ±10% | 舵机个体差异 |
| 零件质量 | ±15% | PLA 密度不均匀 |
| 观测噪声 | 0-0.5° | 无编码器开环漂移 |

效果对比：

| 训练方式 | 仿真成功率 | 真机迁移成功率 |
|---------|:---------:|:-------------:|
| 无域随机化 | 95% | ~40% |
| 有域随机化 | 92% | ~70-80% |

## 6.6 Benchmark 任务定义

平台定义了 7 个标准化评估任务：

| 任务 | 核心目标 | 难度 |
|------|---------|:---:|
| reach | 末端到达目标点 | ★☆☆ |
| push | 推动物体到指定区域 | ★★☆ |
| pick_place | 抓取并放置物体 | ★★★ |
| stack | 堆叠积木 | ★★★ |
| follow | 跟踪移动目标 | ★★☆ |
| gesture | 做出指定手势 | ★★☆ |
| dance | 跟随节奏运动 | ★★★ |

每个任务包含三个要素：奖励函数、初始状态分布、终止条件。
