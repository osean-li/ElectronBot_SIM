# ElectronBot-SIM 教学书 Markdown 页面模板

> 适用于 `docs/bilibili/` 下所有章节子页面的 Markdown 编写规范。
> 支持 `images/` 文件夹下的图片引用，图片路径格式：`images/xxx.png`。
> 每页目标阅读时长 5-12 分钟。

---

## 页面骨架（从上到下）

```
# 章号.节号 标题                           ← H1（与 header 标题一致）
> 📖 上下文回顾 / 本节背景                    ← 引用块，承上启下

核心概念 / 本节要点                          ← 知识卡片
关键术语标签

________________________________________      ← 分隔线

## 正文标题 1                                 ← H2 正文（1-3 个）

正文段落，200-400 字，有逻辑递进。

![图片说明](images/xxx.png)                   ← 配图引用 images/ 文件夹

### 小节标题                                 ← H3 子节

| 列A | 列B | 列C |                          ← 表格
|-----|-----|-----|

```bash
命令代码                                      ← 代码块
```

________________________________________

> 🔑 关键词释义                              ← 术语表
> - **术语**：一句话定义

> 📂 本节涉及的代码文件                       ← 代码索引
> - `文件路径` — 说明

> ⚙️ 实现逻辑                                 ← 逻辑小结
> 1. **关键词**：步骤摘要
> 2. **关键词**：步骤摘要
```

---

## 各区块编写规范

### 1. 页面头部

```markdown
# 2.1 确认环境依赖

> **📖 上下文回顾**：上一节你安装好了系统依赖……现在需要<b>确认 Python 和 MuJoCo 版本</b>。
```

**规则**：
- H1 标题与正文第一个 H2 不能重复。
- 引用块（`>`）承接上一节，引出本节核心问题。
- 必须有一句加粗的核心句。

---

### 2. 图片引用

**规则**：所有图片放在当前章节的 `images/` 文件夹中，使用相对路径引用。

```markdown
![ESP32-S3 UART 接口](images/esp32s3_uart_port.png)

![架构图](images/arch_layer_2.svg)
```

支持的格式：`.png`、`.jpg`、`.svg`、`.gif`。

**命名规范**：
- 架构图：`arch_xxx.svg`
- 硬件照片：`hardware_xxx.jpg`
- 流程/截图：`flow_xxx.png`
- 数据图：`chart_xxx.png`

---

### 3. 知识卡片（三种变体）

```markdown
<!-- concept 卡：解释深层概念 -->
> **💡 核心概念：MuJoCo 运动学链**
> 
> 运动学链（kinematic chain）描述了机器人各关节之间的父子关系……
> - body 是腰部的父节点
> - head 和 arm 是 body 的子节点

<!-- summary 卡：总结本节产出 -->
> **✅ 本节要点**
> 
> - 安装了 Python 3.11 + MuJoCo 3.2.0
> - 成功用 Viewer 加载了 electronbot_scene.xml

<!-- extended 卡：扩展知识 -->
> **📚 扩展知识：FreeCAD vs Fusion360**
> 
> FreeCAD 是开源免费的 CAD 软件……
```

---

### 4. 正文主体

**结构层次**：
- `##` = 章节标题（2-3 个）
- `###` = 小节标题（按需）
- 段落每段 200-400 字，有逻辑递进

**深度标准**：
- 不只说"是什么"，必须解释"为什么"
- 代码标注文件路径：`src/electronbot_sim/env.py`
- 固件对标标注源码位置：`movements.cc:87`
- 变量名用反引号包裹

**代码块格式**：
````markdown
```bash
pip install mujoco gymnasium numpy
python3 -m mujoco.viewer --mjcf=assets/mjcf/electronbot_scene.xml
```
````

**表格格式**：
```markdown
| 工具 | 版本 | 用途 |
|------|:----:|------|
| Python | 3.11 | 运行环境 |
| MuJoCo | 3.2.0 | 物理仿真引擎 |
```

---

### 5. 关键词释义表

```markdown
> **🔑 关键词释义**
> 
> | 术语 | 定义 |
> |------|------|
> | MJCF | MuJoCo 的 XML 模型描述格式 |
> | STL | 三角网格格式，用于 3D 模型的几何表示 |
> | Gymnasium | RL 环境的标准化接口规范 |
```

**规则**：3-6 个术语，简短定义 20-40 字。

---

### 6. 代码位置索引

```markdown
> **📂 本节涉及的代码文件**
> 
> - `assets/mjcf/electronbot.xml` — 机器人模型本体（7 body / 6 joint / 6 actuator）
> - `src/electronbot_sim/env.py` · `ElectronBotEnv.step()` — 执行单步仿真推进
```

**规则**：4-5 条为宜，文件路径 + 函数名 + 简短说明。

---

### 7. 实现逻辑小结

```markdown
> **⚙️ 实现逻辑**
> 
> 1. **建模先行**：从 STEP 图纸出发，导出 STL 网格，填入 XML 模板
> 2. **引擎驱动**：MuJoCo 加载 MJCF，启动物理仿真循环
> 3. **环境封装**：用 Gymnasium 标准接口封装，对接 RL 算法
```

**规则**：
- 4-6 步，每步以加粗关键词开头
- 不能是 H2 标题列表
- 章级页面覆盖全链路，节级页面聚焦本节流程

## 图表类型速查

| 类型 | 何时用 | 示例 |
|------|--------|------|
| 架构图 | 章首节，展示该章的架构层 | `![L2 物理仿真层](images/arch_layer_2.svg)` |
| 硬件照片 | 涉及真机、PCB、焊接、接线 | `![ESP32-S3 主板](images/hardware_pcb.jpg)` |
| 流程/截图 | 多步骤操作过程 | `![FreeCAD 导出步骤](images/flow_freecad_export.png)` |
| 对比表 | 两个以上方案/参数对比 | 用 Markdown 表格 |
| 数据图 | 性能数据或分布 | `![FPS 基准测试](images/chart_fps_benchmark.png)` |

---

## 全局约束

1. **不重复**：H1/H2 不重复，引用块不复读正文首句
2. **有深度**：每句话要么解释"是什么"，要么解释"为什么"
3. **可执行**：所有命令完整、可直接复制
4. **图片本地化**：图片放 `images/`，引用路径 `images/xxx.png`
5. **有对标**：仿真行为对齐真机时，标注固件源码位置
6. **渐进式**：每页从上一页产出出发，引出本节问题
