3.8 仿真稳定性调优（v3.0 更新）

3.8.1 v3.0 稳定性优化

v3.0 通过以下优化大幅提升仿真稳定性：

  1. 坐标本地化：STL 顶点在脚本中完成 mm→m 转换，避免 MuJoCo 自动中心化导致的惯性参数放大

  2. 质量优化：零件质量从 0.5kg 降至 0.01-0.015kg，避免过高质量导致仿真不稳定

  3. 执行器增益：kp 从 20 降至 5，kv 从 1 降至 2，抑制高频振动

  4. 关节阻尼：从 4.0 降至 1.0，平衡响应速度与稳定性

3.8.2 积分器选择

尽管 v3.0 已大幅改善，RK4 积分器仍然是必要的，因为 347 个零件的复杂装配体对数值精度要求极高。

四种积分器的对比测试结果：

积分器 稳定性 备注
implicitfast（默认） ❌ 爆炸 近似隐式欧拉，对极端条件不稳定
Euler ❌ 爆炸 一阶显式，最不稳定
implicit ❌ 爆炸 全隐式仍不稳定
RK4 ✅ 稳定 4 阶龙格-库塔，精度最高，代价为 4 倍计算量

3.8.3 v2.0 稳定性调试过程（历史参考）

v2.0 使用 inline mesh 时遇到的 NaN 爆炸问题（v3.0 已通过坐标本地化解决）：

  1. 第一次加载：执行 python3 -m mujoco.viewer --mjcf=assets/mjcf/electronbot_full_arm.xml，模型显示正常。

  2. 拉动 actuator slider：轻轻拖动 act_body，机器人立刻飞走，屏幕数值变成 NaN。仿真崩溃。

  3. 排查接触参数：怀疑是碰撞检测导致。调了 <geom condim>、friction、margin——没用。

  4. 排查网格质量：用 trimesh 检查 STL 是否有反转法线、重复顶点——网格质量正常。

  5. 排查质量设置：把 geom mass 从 0.005 调到 0.5——没用。

  6. 怀疑阻尼不足：把 damping 从 0.01 调到 1.0——还是崩。

  7. 尝试换积分器：MuJoCo 的 <option integrator> 支持 4 种。逐一测试：

  • implicitfast → 崩

  • Euler → 崩

  • implicit → 崩

  • RK4 → 没崩！

  1. 确认根因：同时把 damping 调到 4.0、armature 调到 0.1，模型完全稳定。

  2. 记录日志：在开发笔记中记下"STL mm 单位 → 惯性放大 10¹² 倍 → implicitfast 扛不住 → 换 RK4 + 提 damping"。

3.8.4 v3.0 参数配置

v3.0 的最终参数配置：

<option timestep="0.002" integrator="RK4" iterations="50"/>

<default>
  <joint damping="1.0" armature="0.01"/>
</default>

<actuator>
  <position name="body_pos" joint="body_joint" kp="5" kv="2"/>
  <position name="head_pos" joint="head_joint" kp="5" kv="2"/>
  <position name="left_pitch_pos" joint="left_pitch_joint" kp="5" kv="2"/>
  <position name="left_roll_pos" joint="left_roll_joint" kp="5" kv="2"/>
  <position name="right_pitch_pos" joint="right_pitch_joint" kp="5" kv="2"/>
  <position name="right_roll_pos" joint="right_roll_joint" kp="5" kv="2"/>
</actuator>

关键参数说明：

  • damping：1.0，平衡响应速度与稳定性（v2.0 为 4.0）

  • armature：0.01，关节惯性（v2.0 为 0.1）

  • kp：5，位置增益（v2.0 为 20，过高导致振荡）

  • kv：2，速度增益（v2.0 为 1，增加阻尼效果）

  • iterations：50，求解器迭代次数

3.8.5 坐标本地化对稳定性的影响

v3.0 的坐标本地化方案对仿真稳定性的关键影响：

  • 避免惯性放大：v2.0 使用 inline mesh 时，MuJoCo 会自动将 mesh 质心移到 body 原点，导致惯性参数放大约 10¹² 倍。v3.0 通过脚本在导出 STL 时完成坐标转换，避免这个问题。

  • 质量分布合理：每个零件质量控制在 0.01-0.015kg，总质量约 3.5kg，符合实际机器人重量分布。

  • 关节轴对齐：坐标本地化确保关节轴与零件几何中心对齐，避免力矩不平衡导致的振荡。
