3.2 三类文件的定位

在开始操作前，需明确三种文件格式在流程中的角色：

文件格式 角色 内容 工具
CAD (.step) 原始设计图纸 347 个零件的精确几何与装配关系（30.5MB） FreeCAD / SolidWorks
STL (.stl) 3D 表面网格 347 个独立零件的三角形顶点坐标，外部引用 cascadio + trimesh 自动提取
MJCF (.xml) 仿真描述文件 刚体层次、关节定义、执行器参数、外部 STL 引用 step_to_mjcf.py 脚本生成

三段转换关系：

CAD（设计意图）→ STL（外形）→ MJCF（运动描述）

3.3 零件分组与运动组确认（v3.0 自动化方案）

3.3.1 自动提取 347 个零件

v3.0 方案使用 scripts/step_to_mjcf.py 自动从 STEP 文件提取所有零件：

python scripts/step_to_mjcf.py

脚本核心逻辑：

  1. 使用 cascadio 将 STEP 转换为 GLB 格式（保留零件边界和色彩）

  2. 使用 trimesh 读取 GLB 场景图，遍历 scene graph

  3. 自动提取 347 个独立零件的 mesh 和 transform

  4. 按名称自动分类到 body/head/neck/shoulder/left_arm/right_arm

  5. 导出每个零件的 STL 到 assets/meshes/step_parts/

3.3.2 自动分类规则

脚本根据零件名称中的关键词自动分类：

分组 关键词匹配 零件数量 运动类型
body body, torso, chest ~80 绕 Z 轴旋转
head head, face, eye ~45 绕 Y 轴俯仰
neck neck ~15 连接头部与身体
shoulder shoulder ~20 肩膀连接件
left_arm left, arm_l ~90 pitch + roll 双关节
right_arm right, arm_r ~97 pitch + roll 双关节

3.3.3 坐标本地化

脚本自动完成坐标转换，使每个零件相对于其父 body 定位：

# 1. 计算所有零件的最低 z 坐标（用于让模型坐地）
min_z = min([p["min_z"] for p in all_parts])
z_shift = -min_z

# 2. 计算每个零件相对于父 body 的偏移
def _localize_parts(parts, parent_world_pos):
    for p in parts:
        stl_path = stl_base / p["stl"]
        mesh = trimesh.load(stl_path)
        # 顶点加 z_shift（让模型坐地）
        mesh.vertices[:, 2] += z_shift
        # 转换为 local 坐标（减去父 body 世界位置）
        mesh.vertices -= parent_world_pos
        local_path = stl_path.with_name(stl_path.stem + "_local.stl")
        mesh.export(local_path)
        p["local_stl"] = local_path.name

关键点：

  • geom.pos = 0：几何体无额外偏移

  • 最终显示位置 = body.pos + geom.pos + localVertex = parent_world_pos + 0 + (worldVertex + z_shift - parent_world_pos) = worldVertex + z_shift

  • 这样确保零件显示在正确位置，且模型最低点接触地面

🔑 关键词释义
FreeCAD开源参数化 CAD 建模软件，支持 Python 脚本和命令行批量导出。
STEPISO 10303 标准 CAD 交换格式，保存精确几何与装配关系。
STL三角网格格式，广泛用于 3D 打印与仿真模型的几何表示。
Inline Mesh将网格顶点数据直接嵌入 MJCF XML 文件中，确保单文件自包含。
RK4四阶 Runge-Kutta 数值积分方法，MuJoCo 默认积分器。
