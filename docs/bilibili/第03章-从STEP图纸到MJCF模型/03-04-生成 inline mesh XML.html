3.7 生成外部 STL 引用 XML（v3.0）

3.7.1 生成脚本核心逻辑

scripts/step_to_mjcf.py 的核心流程：

  1. 使用 cascadio 将 STEP 转换为 GLB 格式（保留零件边界和色彩）

  2. 使用 trimesh 读取 GLB 场景图，遍历 scene graph

  3. 自动提取 347 个独立零件的 mesh 和 transform

  4. 按名称自动分类到 body/head/neck/shoulder/left_arm/right_arm

  5. 导出每个零件的 STL 到 assets/meshes/step_parts/

  6. 坐标本地化（世界坐标→父 body 相对坐标）

  7. 生成外部 STL 引用格式的 MJCF XML

3.7.2 运行脚本

python scripts/step_to_mjcf.py

脚本输出：

  • assets/meshes/step_parts/*.stl：347 个独立零件 STL

  • assets/meshes/step_parts/*_local.stl：347 个本地化后的 STL

  • assets/mjcf/electronbot_step_meters.xml：生成的 MJCF 文件（~103KB）

3.7.3 最终模型结构

<mujoco model="electronbot">
  <compiler angle="radian" autolimits="true"/>
  <option timestep="0.002" integrator="RK4" iterations="50"/>

  <asset>
    <!-- 347 个外部 STL 引用 -->
    <mesh name="body_part001_local" file="../meshes/step_parts/body_part001_local.stl"/>
    <mesh name="body_part002_local" file="../meshes/step_parts/body_part002_local.stl"/>
    <!-- ... 更多零件 ... -->
  </asset>

  <worldbody>
    <body name="base_link" pos="0 0 0">
      <geom type="mesh" mesh="base_part001_local" mass="0.01"/>
      <!-- ... 更多 base 零件 ... -->
      
      <body name="body" pos="0 0 0.03">
        <joint name="body_joint" type="hinge" axis="0 0 1"
               range="-1.5708 1.5708"/>
        <geom type="mesh" mesh="body_part001_local" mass="0.01"/>
        <!-- ... 更多 body 零件 ... -->

        <body name="head" pos="0 0 0.07">
          <joint name="head_joint" type="hinge" axis="0 1 0"
                 range="-0.2618 0.2618"/>
          <geom type="mesh" mesh="head_part001_local" mass="0.01"/>
          <!-- ... 更多 head 零件 ... -->
        </body>

        <body name="left_arm_pitch" pos="-0.018 0 0.065">
          <joint name="left_pitch_joint" type="hinge" axis="0 1 0"
                 range="-1.5708 1.5708"/>
          <body name="left_arm_roll">
            <joint name="left_roll_joint" type="hinge" axis="1 0 0"
                   range="-0.7854 0.7854"/>
            <geom type="mesh" mesh="left_arm_part001_local" pos="0 0.04 0" mass="0.015"/>
            <!-- Y 轴偏移 4cm 避免碰撞 -->
          </body>
        </body>

        <!-- 右臂结构同左臂，pos 方向相反 -->
      </body>
    </body>
  </worldbody>
</mujoco>

3.7.4 Actuator 参数（v3.0 优化）

关节 舵机 kp kv 控制范围 (rad) 对应角度
body_joint SG90 5 2 ±1.57 ±90°
head_joint 2g 5 2 ±0.26 ±15°
left/right_pitch 2g 5 2 ±1.57 ±90°
left/right_roll 2g 5 2 ±0.79 ±45°

v3.0 优化：降低 kp/kv 值（从 30-80 降至 5），避免仿真爆炸。零件质量从 0.005-0.060kg 降至 0.01-0.015kg，提高稳定性。

3.7.5 关节结构优化

v3.0 将 roll 关节放置在独立子 body 中，确保独立运动：

<!-- v2.0：pitch 和 roll 在同一 body -->
<body name="left_arm">
  <joint name="left_pitch_joint" .../>
  <joint name="left_roll_joint" .../>
  <geom .../>
</body>

<!-- v3.0：roll 在子 body 中 -->
<body name="left_arm_pitch">
  <joint name="left_pitch_joint" .../>
  <body name="left_arm_roll">
    <joint name="left_roll_joint" .../>
    <geom .../>
  </body>
</body>

这种结构使 pitch 和 roll 关节可以独立旋转，实现更自然的手臂运动。

🔑 关键词释义
FreeCAD开源参数化 CAD 建模软件，支持 Python 脚本和命令行批量导出。
STEPISO 10303 标准 CAD 交换格式，保存精确几何与装配关系。
STL三角网格格式，广泛用于 3D 打印与仿真模型的几何表示。
Inline Mesh将网格顶点数据直接嵌入 MJCF XML 文件中，确保单文件自包含。
RK4四阶 Runge-Kutta 数值积分方法，MuJoCo 默认积分器。
