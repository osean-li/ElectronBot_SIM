3.9 场景管理（v3.0 更新）

3.9.1 v3.0 单文件方案

v3.0 不再拆分场景文件，而是使用单文件方案：

  • 唯一模型文件：assets/mjcf/electronbot_step_meters.xml（~103KB）

  • 外部 STL 引用：347 个零件独立存储在 assets/meshes/step_parts/

  • MCP 服务器控制：通过 --render human 参数启动可视化窗口

# 启动 MCP 服务器（带可视化）
python3 -m electronbot_sim.mcp_server --render human

# 启动 MCP 服务器（无可视化，仅 WebSocket）
python3 -m electronbot_sim.mcp_server

3.9.2 相机配置

v3.0 的相机配置确保模型完整显示在地面：

# 在 mcp_server.py 中
camera_distance = 0.3  # 米
lookat_height = 0.05   # 米

这些参数确保：

  • 模型最低点接触地面（通过 z_shift 计算）
  • 相机距离适中，既能看到全貌又能看清细节
  • lookat 高度略高于地面，避免视角过低

3.9.3 v2.0 场景拆分问题（历史参考）

v2.0 使用 inline mesh 时遇到的场景拆分问题（v3.0 已通过外部引用和坐标本地化解决）：

问题 根因 v3.0 解决方案
XML 文件过大（>10MB） 347 个零件的 inline mesh 数据 外部 STL 引用，XML 仅 ~103KB
零件位置不对 MuJoCo 自动中心化 mesh 坐标本地化（世界坐标→父 body 相对坐标）
模型悬空 未计算最低点偏移 脚本自动计算 z_shift，确保模型坐地
零件断开 缩放 mesh 后 body 位置未同步 STL 顶点已在脚本中完成 mm→m 转换
棋盘格地面不可见 模型尺寸畸变（70m 高） 模型尺寸正确（~0.2m 高），相机距离 0.3m

3.9.4 v3.0 场景管理流程

v3.0 的场景管理流程：

  1. 生成模型文件：运行 python scripts/step_to_mjcf.py，自动生成 electronbot_step_meters.xml 和 347 个 STL 文件

  2. 启动 MCP 服务器：python3 -m electronbot_sim.mcp_server --render human

  3. 自动加载可视化：MuJoCo 窗口自动打开，相机自动适配模型尺寸

  4. WebSocket 控制：通过 ws://localhost:8080/ws 发送动作指令

优势：

  • 无需手动管理场景文件
  • 相机参数自动适配，无需手动调整
  • 模型文件简洁，易于版本管理
  • 通过 MCP 服务器参数控制渲染模式

3.10 本章小结（v3.0 更新）

本章从 STEP 原始 CAD 出发，经过自动化零件提取、坐标本地化、外部 STL 引用、稳定性调优，最终得到完整的 MJCF 模型文件。

v3.0 与设计文档的主要差异：

项目 设计文档（v2.0） v3.0 实现
零件提取 FreeCAD 手动导出 24 零件 cascadio + trimesh 自动提取 347 零件
STL 导出策略 按 5 运动组合并导出 每个零件独立 STL
MJCF 方案 Inline Mesh（单文件 >10MB） 外部 STL 引用（XML ~103KB）
坐标处理 MuJoCo 自动中心化 脚本坐标本地化（世界坐标→父 body 相对坐标）
积分器 implicitfast RK4（仍然必要）
场景管理 2 个独立场景文件 单文件方案，MCP 服务器控制渲染
模型尺寸 mm→m 未缩放，模型高 70m 脚本自动转换，模型高 ~0.2m，相机距 0.3m
零件质量 未指定 0.01-0.015kg，总质量 ~3.5kg
执行器增益 kp=20, kv=1 kp=5, kv=2（抑制振荡）

v3.0 关键改进

  1. 自动化提取：从 24 个手动导出零件到 347 个自动提取零件，大幅提高建模精度
  2. 坐标本地化：避免 MuJoCo 自动中心化导致的惯性参数放大 10¹² 倍
  3. 外部引用：XML 文件从 >10MB 降至 ~103KB，易于版本管理和编辑
  4. 质量优化：零件质量控制在合理范围，避免仿真不稳定
  5. 单文件方案：无需场景拆分，通过 MCP 服务器参数控制渲染模式
🔑 关键词释义
FreeCAD开源参数化 CAD 建模软件，支持 Python 脚本和命令行批量导出。
STEPISO 10303 标准 CAD 交换格式，保存精确几何与装配关系。
STL三角网格格式，广泛用于 3D 打印与仿真模型的几何表示。
Inline Mesh将网格顶点数据直接嵌入 MJCF XML 文件中，确保单文件自包含。
RK4四阶 Runge-Kutta 数值积分方法，MuJoCo 默认积分器。
