6.1 固件版本选择与烧录

阅读约 8 分钟 ElectronBot · ESP32-S3 · v2.2.6-2
核心关键词 v2.2.6-2WebSocketesptool烧录ESP32-S3在线烧录
上下文回顾
上一章你实现了 MCP 协议桥接层，现在需要在真机上验证 MCP 通信和舵机控制是否与仿真一致。 但在此之前，真机固件必须升级到支持 WebSocket 直连的版本。
关键发现：v2.2.6-2 的 WebSocket Server
原设计文档假设"真机无 WebSocket Server，必须通过云端 API 透传"。
但 2026-06-16 发布的 v2.2.6-2 已内置 ws://<IP>:8080/ws WebSocket 调试服务！
这意味着你可以直接从 Python 端通过 WebSocket 控制真机，延迟从 200-500ms 降至 <10ms。

方法一：在线一键烧录（最简单）

  1. 用 Chrome/Edge 浏览器打开 https://electronbot.tech/docs/downloads
  2. 固件版本选择：v2.2.6-2
  3. 点击"在线烧录"按钮
  4. 浏览器弹出 USB 串口选择框，选择 ElectronBot 对应 COM 口
  5. 等待烧录完成，自动重启
提示：下载模式
若浏览器无法识别设备，按住主板 BOOT 按钮再上电进入下载模式，然后重试。

方法二：esptool 命令行（推荐 Linux/macOS 用户）

# 安装 esptool
pip install esptool

# 下载固件
wget https://electronbot.tech/assets/files/electronbot2.2.6-2-8777ea4d7e4cb1fe838db75ac1b55ecc.bin

# 确认串口
ls /dev/ttyUSB*  # 或 /dev/ttyACM*

# 烧录
esptool.py --chip esp32s3 --port /dev/ttyUSB0 --baud 921600 \
  --before default_reset --after hard_reset \
  write_flash -z --flash_mode dio --flash_size 16MB \
  0x0 electronbot2.2.6-2-8777ea4d7e4cb1fe838db75ac1b55ecc.bin

验证烧录成功

# 1. 用串口监控确认配网
minicom -D /dev/ttyUSB0 -b 115200
# 应看到 Wi-Fi 连接日志和 IP 地址获取

# 2. 测试 WebSocket 直连（假设真机 IP = 192.168.1.100）
websocat ws://192.168.1.100:8080/ws
> {"type":"mcp","payload":{"jsonrpc":"2.0","method":"tools/call","params":{"name":"self.electron.get_status","arguments":{}},"id":1}}
< {"type":"mcp","payload":{"jsonrpc":"2.0","id":1,"result":{"content":[{"type":"text","text":"idle"}],"isError":false}}}

# 3. 测试手部动作
> {"type":"mcp","payload":{"jsonrpc":"2.0","method":"tools/call","params":{"name":"self.electron.hand_action","arguments":{"action":3,"hand":3,"steps":2,"speed":600}},"id":2}}
# → 机器人应挥手

常见问题

问题原因解决
Failed to connect to ESP32-S3未进入下载模式按住 BOOT 按钮再上电
Invalid head of packet波特率过高降至 115200
WebSocket Connection refused固件不是 v2.2.6-2升级到 v2.2.6-2
连接后立即断开非 MCP 格式消息使用 {"type":"mcp","payload":{...}} 格式
本节逻辑小结
  1. v2.2.6-2 是验证真机 MCP 通信的最低版本（WebSocket 直连）
  2. 在线一键烧录最简单，esptool 适合自动化脚本
  3. 烧录后用 websocat 验证 WebSocket 通信正常
