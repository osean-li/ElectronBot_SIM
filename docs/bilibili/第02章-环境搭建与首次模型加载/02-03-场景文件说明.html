2.4 场景文件说明

本书所有操作统一使用 electronbot_scene.xml 作为主文件。场景文件通过 <include file="electronbot.xml"/> 引用机器人模型（见代码清单 2-1），自身仅管理棋盘格地面、渐变天空和灯光等视觉效果，场景元素可通过修改 <visual>、<asset> 及 <worldbody> 中的参数自由调整。

<!-- 代码清单 2-1 electronbot_scene.xml 结构 -->
<mujoco model="electronbot scene">
  <include file="electronbot.xml"/>

  <visual>
    <headlight diffuse="0.8 0.8 0.8" ambient="0.4 0.4 0.4" specular="0.1 0.1 0.1"/>
    <rgba haze="0.15 0.25 0.35 1"/>
    <global azimuth="-130" elevation="-20" offwidth="960" offheight="720"/>
  </visual>

  <asset>
    <texture type="skybox" builtin="gradient"
             rgb1="0.3 0.5 0.7" rgb2="0 0 0" width="512" height="3072"/>
    <texture type="2d" name="groundplane" builtin="checker" mark="edge"
             rgb1="0.50 0.60 0.72" rgb2="0.06 0.13 0.22"
             markrgb="0.9 0.9 0.9" width="300" height="300"/>
  </asset>

  <worldbody>
    <light pos="0 0 1.5" dir="0 0 -1" directional="true"/>
    <geom name="floor" type="plane" material="groundplane"/>
  </worldbody>
</mujoco>

assets/mjcf/ 目录下同时存在 electronbot_full_arm.xml（模型与场景未拆分的单文件版本），可作为参考，但后续课程不再使用。

2.5 模型结构快速检查

除可视化验证外，亦可通过 Python 脚本快速获取模型的结构化信息：

python3 -c "
import mujoco
m = mujoco.MjModel.from_xml_path('assets/mjcf/electronbot_scene.xml')
print(f'刚体数: {m.nbody}')
print(f'关节数: {m.njnt}')
print(f'执行器数: {m.nu}')
print(f'几何体数: {m.ngeom}')
"

预期输出：刚体 7，关节 6，执行器 6，几何体 6。

2.6 本章小结

本章完成了环境搭建与首次模型加载验证。实际实现与概要设计文档的差异总结如下：

项目 设计文档 v1.2 实际实现
输出文件名 electronbot_mesh.xml electronbot_scene.xml（场景文件）
electronbot.xml（模型本体）
STL 导出方式 24 零件分别导出 按运动组合并导出为 5 组
积分器 implicitfast RK4（稳定性问题）
场景文件 2 个独立场景文件 统一为 electronbot_scene.xml
关节命名 joint_body / joint_head / joint_lp body_joint / head_joint / left_pitch_joint
