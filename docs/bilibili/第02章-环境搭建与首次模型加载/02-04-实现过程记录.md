## 2.7 实现过程记录

### 2.7.1 所用工具和版本

| 工具 | 版本 | 用途 |
|------|:----:|------|
| Python | 3.11 | 运行环境和生成脚本 |
| MuJoCo | 3.2.0 | 物理仿真引擎和 Viewer |
| mujoco (pip) | 3.2.0 | Python binding |

### 2.7.2 执行命令汇总

```bash
# 1. 安装依赖
pip install mujoco gymnasium numpy scipy opencv-python trimesh

# 2. 验证安装
python3 -c "import mujoco; print(mujoco.__version__)"

# 3. 启动 Viewer
python3 -m mujoco.viewer --mjcf=assets/mjcf/electronbot_scene.xml

# 4. 命令行检查模型结构
python3 -c "
import mujoco
m = mujoco.MjModel.from_xml_path('assets/mjcf/electronbot_scene.xml')
print(f'nbody={m.nbody}, njnt={m.njnt}, nu={m.nu}, ngeom={m.ngeom}')
"
```

### 2.7.3 过程中踩到的坑

1. **Viewer 窗口出不来**：在 SSH 远程终端运行时报错 `GLFWError: DISPLAY environment variable is missing`。因为远程没有显示器。解决方案在后面章节会统一处理（详见 L5 传感器层中关于 EGL headless 渲染的讨论）。

> **🔑 关键词释义**
>
> | 术语 | 定义 |
> |------|------|
> | STL | 三角网格格式，广泛用于 3D 打印与仿真模型的几何表示。 |
> | Viewer | MuJoCo 内置的交互式 3D 查看器，支持拖拽视角、关节滑块控制和实时仿真状态显示。 |

> **📂 本节涉及的代码文件**
>
> - `assets/mjcf/electronbot_scene.xml` — 场景主文件，本章所有操作的入口
> - `assets/mjcf/electronbot_full_arm.xml` — inline mesh 版 MJCF 模型（备用参考）

> **⚙️ 实现逻辑**
>
> 1. **安装依赖**：pip 一键安装 mujoco 及配套科学计算包。
> 2. **验证安装**：通过 Python import 确认 mujoco 包可正常导入，打印版本号。
> 3. **启动 Viewer**：通过 `mujoco.viewer` 模块加载 electronbot_scene.xml，在本地窗口中查看模型。
> 4. **命令行检查**：用 MjModel.from_xml_path() 程序化验证模型结构（body/joint/actuator/geom 数量）。
> 5. **踩坑记录**：SSH 远程环境缺少 DISPLAY 环境变量导致 Viewer 无法启动，后续通过 EGL headless 模式解决。
