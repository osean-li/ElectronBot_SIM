上一节你搭建好了环境，验证了 MuJoCo 能导入。现在你要真正加载 ElectronBot 的 MJCF 模型——这意味着 MuJoCo 需要解析 electronbot_full_arm.xml 中的 7 个 body（运动学链）、6 个 hinge 关节、6 个 position actuator 和 inline mesh 数据。加载成功后，你会得到一个可拖拽视角的 3D 窗口，右侧 Control 面板的 6 个滑块分别控制机器人的身体、头部、左右臂。本节的核心任务是验证运动学链的完整性：每个关节是否能独立运动、父关节是否带动子关节、外壳零件是否错误地跟随了不该跟随的关节。

⚠️ 视觉比例偏大：首次加载你可能会发现机器人"身高 70 米"。这是因为 CAD 图纸以毫米为单位，而 MuJoCo 默认以米为单位解析 STL 网格——所以模型被放大了 1000 倍。这不影响物理仿真的正确性（质量、惯量、关节力矩都是独立指定的），第三章将详述原因和处理方案。

核心概念：6 关节运动学链

ElectronBot 的机械结构形成了一条从底座到指尖的树形运动学链：

  • base_link（固定）→ body_joint（绕 Z 轴旋转 ±90°）→ head_joint（绕 Y 轴俯仰 ±15°）+ left_pitch + left_roll（左臂 2 自由度）+ right_pitch + right_roll（右臂 2 自由度）
  • 你要验证的关键性质：身体旋转时头部和双臂必须随动（它们挂在 body link 下）；手臂运动时身体侧外壳不能随之转动（外壳挂在 body link 下，不属于 arm link）。

2.3 启动 Viewer，加载模型

环境就绪后，一行命令启动 MuJoCo 内置的交互式查看器：

python3 -m mujoco.viewer --mjcf=assets/mjcf/electronbot_full_arm.xml

约 2 秒后弹出一个 OpenGL 窗口。你会看到：银色双臂机器人站在棋盘格地面上，蓝色渐变天空背景，右侧是 Control 面板。用鼠标左键拖拽旋转视角，滚轮缩放，中键平移。

electronbot_full_arm.xml 是推荐使用的入口文件——它使用 inline mesh 将所有 STL 网格嵌入 XML 中（~1.7MB 单文件），零外部路径依赖，跨平台无差异。另外两个场景文件 scene_single.xml 和 scene_dual.xml 通过 <include file="electronbot.xml"/> 引入公共模型，用于后续的特定任务（见下一节）。

2.4 验证 6 个关节——逐项检查运动学链

Viewer 右侧 Control 面板的 Actuators 区域列出了 6 个滑块。每个滑块对应 MJCF 中定义的一个 position actuator，它的名字揭示了它所驱动的关节。逐一拖动，按以下标准验证：

滑块名称MuJoCo 关节MCP 代号运动轴机械范围验证要点
act_bodybody_jointb绕 Z 轴旋转±90°身体旋转 → 头部+双臂随动；底座不动
act_headhead_jointh绕 Y 轴俯仰±15°头部俯仰 → 双臂不受影响
act_right_pitchright_pitch_jointrp绕 Y 轴旋转±90°右臂前后摆 → 外壳不动
act_right_rollright_roll_jointrr绕 X 轴旋转±45°右臂侧摆 → 外壳不动
act_left_pitchleft_pitch_jointlp绕 Y 轴旋转±90°左臂前后摆 → 外壳不动
act_left_rollleft_roll_jointlr绕 X 轴旋转±45°左臂侧摆 → 外壳不动

MCP 代号这列你现在可能觉得多余——但它在第四章和第五章会反复出现。MCP 协议用两字母缩写（rp/rr/lp/lr/b/h）指代 6 个舵机，它们直接映射到 MuJoCo 的 6 个 actuator。这个映射是 Sim2Real 零修改部署的基础——仿真里驱 act_right_pitch，真机上就是给舵机 rp 发 PWM。

验证步骤（按我实际操作的顺序）

  1. 先拖 act_body：身体绕 Z 轴旋转。如果头部和双臂没有跟着转——说明运动学链的父子关系有 bug，body link 没有正确地作为 head/arm link 的父节点。
  2. 再拖 act_right_pitch + act_right_roll：右臂前后/侧向摆动。如果身体侧外壳跟着动了——说明 035/036 零件被错误地归入了 arm 组而非 body 组。这是第三章"零件分组"中最容易犯的错误。
  3. 再拖 act_left_pitch + act_left_roll：左臂同样验证。左右臂的零件数量和编号不同（左 6 件，右 7 件），但关节结构和范围完全对称。
  4. 最后拖 act_head：头部的俯仰范围只有 ±15°，滑块拖动时动作幅度很小——这是正确的，ElectronBot 原版设计的头部活动范围就是 75°~105°（±15° 相对于中心 90°）。
  5. 快速拖动：把滑块快速拖到极限值，观察是否有抖动或 NaN 崩溃。如果没有——说明关节阻尼和 actuator kp/kv 参数在初始值下是稳定的。第三章会详细调优这些参数。

命令行快速检查（替代 Viewer）

如果你在无头服务器上、或者想用脚本自动验证，下面的 Python 代码可以直接读取模型结构并输出关节信息——不需要 GUI：

import mujoco
model = mujoco.MjModel.from_xml_path("assets/mjcf/electronbot_full_arm.xml")
print(f"刚体数: {model.nbody}, 关节数: {model.njnt}, 执行器数: {model.nu}")
for i in range(model.nu):
    name = mujoco.mj_id2name(model, mujoco.mjtObj.mjOBJ_ACTUATOR, i)
    jnt_id = model.actuator_trnid[i][0]
    jnt_name = mujoco.mj_id2name(model, mujoco.mjtObj.mjOBJ_JOINT, jnt_id)
    print(f"  actuator[{i}]={name} → joint={jnt_name}")

输出应确认：7 body、6 joint、6 actuator，且每个 actuator 正确映射到对应的 joint。

🔑 关键词释义
actuatorMJCF 中的执行器定义，驱动一个关节运动。本项目使用 position actuator（位置控制），kp/kv 参数按舵机规格校准。
运动学链从固定底座（base_link）到末端（指尖）的 body→joint→body 层级树。父 body 的运动自动传递到子 body。
qpos / ctrlqpos 是 MuJoCo 的内部关节位置数组（弧度），ctrl 是写入 actuator 的控制信号。Viewer 滑块操作本质上在改 ctrl。
单位缩放CAD（毫米）→ STL（毫米坐标）→ MuJoCo（默认米），导致视觉放大 1000 倍。不影响物理仿真，因为质量和惯量独立指定。
📂 模型结构与验证脚本
assets/mjcf/electronbot_full_arm.xml— inline mesh 版 MJCF，7 body / 6 hinge joint / 6 position actuator
assets/mjcf/electronbot.xml— 公共模型文件（被 scene 文件 include 引用）
scripts/validate_model.py · validate_model()— 批量验证刚体/关节/执行器数量与命名
mujoco.mj_id2name()— MuJoCo C API 的 Python 绑定，按 ID 查询 actuator/joint 名称
⚙️ 验证逻辑
  1. 加载模型：mjModel.from_xml_path() 解析 MJCF，构建运动学树和物理参数。
  2. 检查运动学链：逐一拖动 6 个 actuator，确认父子 body 的随动关系正确（身体带手臂、手臂不带外壳）。
  3. 验证关节范围：每个关节的机械限位是否与真机设计一致（身体 ±90°、头部 ±15°、手臂 pitch ±90°/roll ±45°）。
  4. 确认无 NaN：快速拖动不引发数值爆炸——说明初始阻尼和 kp/kv 参数稳定。
