# 2.4 场景文件说明

> **📖 本节背景**：上一节你加载了 electronbot_full_arm.xml 并验证了 6 个关节的运动学链。现在需要理解 `assets/mjcf/` 下多个文件的分工——场景文件通过 `<include file="electronbot.xml"/>` 引用机器人模型（见代码清单 2-1），自身仅管理棋盘格地面、渐变天空和灯光等视觉效果，场景元素可通过修改 `<visual>`、`<asset>` 及 `<worldbody>` 中的参数自由调整。理解这个**模型与场景分离**的架构，是后续所有仿真操作的基础。

> **💡 核心概念：MJCF 的场景-模型分离架构**
>
> MJCF 支持 `<include>` 标签实现文件级复用。本书采用的标准架构是：
> - **electronbot.xml**：纯模型定义（body/joint/geom/actuator），不含任何视觉环境元素
> - **scene_*.xml**：场景入口文件，include 模型后添加地面、天空、灯光
> 这种分离的好处是：同一个模型可以被单臂、双臂、头部特写等不同场景复用，无需复制模型定义。修改模型时只需改 electronbot.xml 一处，所有场景自动生效。

**核心关键词**：`MJCF include` `scene文件` `worldbody` `visual` `asset` `inline mesh` `geom` `设计vs实现`

## 2.4 场景文件说明

本书所有操作统一使用 `electronbot_scene.xml` 作为主文件。场景文件通过 `<include file="electronbot.xml"/>` 引用机器人模型（见代码清单 2-1），自身仅管理棋盘格地面、渐变天空和灯光等视觉效果，场景元素可通过修改 `<visual>`、`<asset>` 及 `<worldbody>` 中的参数自由调整。

```xml
<!-- 代码清单 2-1 electronbot_scene.xml 结构 -->
<mujoco model="electronbot scene">
  <include file="electronbot.xml"/>

  <visual>
    <headlight diffuse="0.8 0.8 0.8" ambient="0.4 0.4 0.4" specular="0.1 0.1 0.1"/>
    <rgba haze="0.15 0.25 0.35 1"/>
    <global azimuth="-130" elevation="-20" offwidth="960" offheight="720"/>
  </visual>

  <asset>
    <texture type="skybox" builtin="gradient"
             rgb1="0.3 0.5 0.7" rgb2="0 0 0" width="512" height="3072"/>
    <texture type="2d" name="groundplane" builtin="checker" mark="edge"
             rgb1="0.50 0.60 0.72" rgb2="0.06 0.13 0.22"
             markrgb="0.9 0.9 0.9" width="300" height="300"/>
  </asset>

  <worldbody>
    <light pos="0 0 1.5" dir="0 0 -1" directional="true"/>
    <geom name="floor" type="plane" material="groundplane"/>
  </worldbody>
</mujoco>
```

`assets/mjcf/` 目录下同时存在 `electronbot_full_arm.xml`（模型与场景未拆分的单文件版本），可作为参考，但后续课程不再使用。

## 2.5 模型结构快速检查

除可视化验证外，亦可通过 Python 脚本快速获取模型的结构化信息：

```bash
python3 -c "
import mujoco
m = mujoco.MjModel.from_xml_path('assets/mjcf/electronbot_scene.xml')
print(f'刚体数: {m.nbody}')
print(f'关节数: {m.njnt}')
print(f'执行器数: {m.nu}')
print(f'几何体数: {m.ngeom}')
"
```

预期输出：刚体 7，关节 6，执行器 6，几何体 6。

## 2.6 本章小结

本章完成了环境搭建与首次模型加载验证。实际实现与概要设计文档的差异总结如下：

| 项目 | 设计文档 v1.2 | 实际实现 |
|------|-------------|---------|
| 输出文件名 | `electronbot_mesh.xml` | `electronbot_scene.xml`（场景文件）、`electronbot.xml`（模型本体） |
| STL 导出方式 | 24 零件分别导出 | 按运动组合并导出为 5 组 |
| 积分器 | implicitfast | RK4（稳定性问题） |
| 场景文件 | 2 个独立场景文件 | 统一为 `electronbot_scene.xml` |
| 关节命名 | joint_body / joint_head / joint_lp | body_joint / head_joint / left_pitch_joint |

> **🔑 关键词释义**
>
> | 术语 | 定义 |
> |------|------|
> | STL | 三角网格格式，广泛用于 3D 打印与仿真模型的几何表示。 |
> | MJCF include | MuJoCo XML 的文件级复用机制，允许场景文件引入公共模型定义，实现模型与场景分离。 |
> | inline mesh | MJCF 的特性，将 STL 网格的顶点/面数据直接嵌入 XML 文本中，实现单文件零外部依赖。 |

> **📂 本节涉及的代码文件**
>
> - `assets/mjcf/electronbot_scene.xml` — 场景主文件，include 模型 + 定义视觉环境
> - `assets/mjcf/electronbot.xml` — 纯机器人模型定义（被 scene 文件 include）
> - `assets/mjcf/scene_single.xml` — 右臂场景入口
> - `assets/mjcf/scene_dual.xml` — 双臂场景入口

> **⚙️ 实现逻辑**
>
> 1. **模型与场景分离**：electronbot.xml 定义纯模型，scene_*.xml 通过 include 引用并添加视觉环境。
> 2. **代码清单 2-1**：electronbot_scene.xml 的完整结构——visual 控制渲染参数，asset 定义天空和地面纹理，worldbody 管理灯光和地面几何体。
> 3. **模型结构检查**：一行 Python 命令验证 body/joint/actuator/geom 数量，确保 MJCF 解析正确。
> 4. **设计 vs 实现对比**：文件名、STL 导出方式、积分器、关节命名的实际选择与设计文档的差异及原因。
