# 第二章 环境搭建与首次模型加载

## 2.1 章节目标

本章从概要设计文档的 Layer 1 出发，目标是在 30 分钟内从零搭建运行环境，并在 MuJoCo Viewer 中成功加载 ElectronBot 模型，验证 6 个关节的基本运动。

## 2.2 确认环境依赖

### 2.2.1 项目目录结构

首先检查项目根目录下的 `assets/` 目录，该目录包含后续所有操作所需的核心资源文件：

| 子目录 | 内容 | 说明 |
|--------|------|------|
| `assets/cad/` | `ElectronBot.step`、`cadelectron.FCStd` | 原始 CAD 图纸，30.5MB，24 个零件 |
| `assets/meshes/` | 5 个合并后的 STL 文件 | 按运动组合并导出的 3D 网格 |
| `assets/mjcf/` | `electronbot_scene.xml`（推荐）<br>`electronbot.xml`（模型本体）<br>`electronbot_full_arm.xml`（单文件版） | MJCF 仿真模型文件 |

### 2.2.2 环境验证

本项目基于 Python 3.11 及 MuJoCo 3.x。按以下命令确认环境就绪：

```bash
python3 --version
# 输出应为 Python 3.11.x

python3 -c "import mujoco; print(mujoco.__version__)"
```

若 MuJoCo 未安装，执行：

```bash
pip install mujoco
```

### 2.2.3 我的环境搭建过程

第一次搭建环境时，我走的流程如下：

1. **确认本机 Python 版本**。通过 `python3 --version` 确认是 3.11。如果版本低于 3.10，MuJoCo 的某些 binding 可能不可用。
2. **创建虚拟环境**。用 conda 创建了一个名为 `ebotsim` 的独立环境，避免和系统 Python 包冲突：
   ```bash
   conda create -n ebotsim python=3.11 -y
   conda activate ebotsim
   ```
3. **安装 MuJoCo 和核心依赖**。MuJoCo 的 pip 包名就是 `mujoco`，版本 >= 3.2.0。一次性装齐后续需要的库：
   ```bash
   pip install mujoco gymnasium numpy scipy opencv-python trimesh
   ```
4. **验证导入**。跑 `python3 -c "import mujoco; print(mujoco.__version__)"`，输出 `3.2.0` 时才放心。
5. **查看项目目录**。`ls assets/` 确认 cad、meshes、mjcf 三个子目录都存在，文件没有被遗漏。

## 2.3 首次加载模型

### 2.3.1 使用 MuJoCo Viewer

执行以下命令启动 MuJoCo 内置的交互式查看器：

```bash
python3 -m mujoco.viewer --mjcf=assets/mjcf/electronbot_scene.xml
```

Viewer 加载成功后，可观察到完整的 ElectronBot 3D 模型，包含底座、身体、头部及左右双臂。`electronbot_scene.xml` 通过 `<include file="electronbot.xml"/>` 引入机器人模型，同时提供了棋盘格地面、渐变天空及灯光等场景元素。后续所有操作均以该文件为主。

> **注**：模型视觉比例偏大（直立高度约 70 米），这是由 STL 毫米单位被 MuJoCo 按米解析所致。此问题不影响物理仿真的正确性，第三章将详述原因及处理方案。

### 2.3.2 验证关节运动

Viewer 右侧 Control 面板列出了 6 个 actuator 滑块，分别对应 6 个关节。逐一拖动验证：

| 滑块名称 | 对应关节 | 运动类型 | 范围 |
|----------|----------|----------|------|
| act_body | body_joint | 绕 Z 轴旋转 | ±90° |
| act_head | head_joint | 绕 Y 轴俯仰 | ±15° |
| act_left_pitch | left_pitch_joint | 绕 Y 轴旋转 | ±90° |
| act_left_roll | left_roll_joint | 绕 X 轴旋转 | ±45° |
| act_right_pitch | right_pitch_joint | 绕 Y 轴旋转 | ±90° |
| act_right_roll | right_roll_joint | 绕 X 轴旋转 | ±45° |

验证标准：
- 6 个关节均可独立拖动，无卡死
- 身体旋转时头部与双臂随动
- 手臂运动时身体侧外壳保持静止
- 无 NaN 崩溃，仿真稳定

### 2.3.3 我的首次加载过程

第一次加载模型时，我做了以下操作：

1. **打开 Viewer**：执行 `python3 -m mujoco.viewer --mjcf=assets/mjcf/electronbot_scene.xml`，大约 2 秒后窗口弹出，机器人出现在视图中。
2. **旋转视角**：按住鼠标左键拖拽旋转，确认模型完整——底座在地面、身体在底座上、头部在身体上、左右臂对称挂在身体两侧。
3. **逐个拖动滑块**：在 Control 面板中展开 Actuators，逐一拖动 6 个滑块：
   - 先拖 `act_body`，身体绕 Z 轴左转右转，头部和手臂随动
   - 再拖 `act_left_pitch`，左臂前后摆动，确认侧外壳不动
   - 再拖 `act_head`，头部上下俯仰，幅度较小（±15°）
4. **发现的问题**：模型目测高约 70 米，相机需要拉很远才能看到全貌。另外拖动滑块过快时模型会抖动一下。这两点不影响后续验证，记录下来等第三章解决。
5. **命令行快速检查**：退出 Viewer，用 Python 脚本检查模型结构参数，确认刚体数、关节数、执行器数。

## 2.4 场景文件说明

本书所有操作统一使用 `electronbot_scene.xml` 作为主文件。场景文件通过 `<include file="electronbot.xml"/>` 引用机器人模型（见代码清单 2-1），自身仅管理棋盘格地面、渐变天空和灯光等视觉效果，场景元素可通过修改 `<visual>`、`<asset>` 及 `<worldbody>` 中的参数自由调整。

```xml
<!-- 代码清单 2-1 electronbot_scene.xml 结构 -->
<mujoco model="electronbot scene">
  <include file="electronbot.xml"/>
  
  <visual>
    <headlight diffuse="0.8 0.8 0.8" ambient="0.4 0.4 0.4" specular="0.1 0.1 0.1"/>
    <rgba haze="0.15 0.25 0.35 1"/>
    <global azimuth="-130" elevation="-20" offwidth="960" offheight="720"/>
  </visual>
  
  <asset>
    <texture type="skybox" builtin="gradient"
             rgb1="0.3 0.5 0.7" rgb2="0 0 0" width="512" height="3072"/>
    <texture type="2d" name="groundplane" builtin="checker" mark="edge"
             rgb1="0.50 0.60 0.72" rgb2="0.06 0.13 0.22"
             markrgb="0.9 0.9 0.9" width="300" height="300"/>
  </asset>
  
  <worldbody>
    <light pos="0 0 1.5" dir="0 0 -1" directional="true"/>
    <geom name="floor" type="plane" material="groundplane"/>
  </worldbody>
</mujoco>
```

`assets/mjcf/` 目录下同时存在 `electronbot_full_arm.xml`（模型与场景未拆分的单文件版本），可作为参考，但后续课程不再使用。

## 2.5 模型结构快速检查

除可视化验证外，亦可通过 Python 脚本快速获取模型的结构化信息：

```bash
python3 -c "
import mujoco
m = mujoco.MjModel.from_xml_path('assets/mjcf/electronbot_scene.xml')
print(f'刚体数: {m.nbody}')
print(f'关节数: {m.njnt}')
print(f'执行器数: {m.nu}')
print(f'几何体数: {m.ngeom}')
"
```

预期输出：刚体 7，关节 6，执行器 6，几何体 6。

## 2.6 本章小结

本章完成了环境搭建与首次模型加载验证。实际实现与概要设计文档的差异总结如下：

| 项目 | 设计文档 v1.2 | 实际实现 |
|------|-------------|---------|
| 输出文件名 | `electronbot_mesh.xml` | `electronbot_scene.xml`（场景文件）<br>`electronbot.xml`（模型本体） |
| STL 导出方式 | 24 零件分别导出 | 按运动组合并导出为 5 组 |
| 积分器 | implicitfast | RK4（稳定性问题） |
| 场景文件 | 2 个独立场景文件 | 统一为 `electronbot_scene.xml` |
| 关节命名 | joint_body / joint_head / joint_lp | body_joint / head_joint / left_pitch_joint |

下一章将从上游开始，完整演示从 STEP 原始 CAD 到 MJCF 模型的生成流程。

### 2.7.3 过程中踩到的坑

1. **conda 环境未激活直接跑脚本**：报了 ModuleNotFoundError，因为没有在正确的环境里。提醒自己每次新开终端先 `conda activate ebotsim`。
