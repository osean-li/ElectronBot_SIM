6.2 源码编译与自定义

阅读约 10 分钟 ESP-IDF · 编译 · OTA · 自定义
核心关键词 ESP-IDF v5.1idf.pymenuconfig编译xiaozhi-esp32
上下文回顾
上一节你通过预编译固件快速烧录了 v2.2.6-2。本节介绍如何从源码编译固件，适用于需要修改 Wi-Fi 配置、调整舵机参数、或适配不同板型的场景。

环境准备（Ubuntu 22.04）

# 系统依赖
sudo apt update
sudo apt install -y \
    git wget flex bison gperf \
    python3 python3-pip python3-venv python3-dev \
    cmake ninja-build ccache \
    libffi-dev libssl-dev libusb-1.0-0-dev \
    dfu-util

# 串口权限
sudo usermod -a -G dialout,plugdev $USER
# 重新登录生效

安装 ESP-IDF v5.1

# 克隆 ESP-IDF v5.1（xiaozhi-esp32 使用的版本）
git clone -b v5.1 --recursive https://github.com/espressif/esp-idf.git ~/esp-idf-v5.1

# 安装工具链
cd ~/esp-idf-v5.1
./install.sh esp32s3

# 激活环境（每次编译前）
source ~/esp-idf-v5.1/export.sh
提示：永久激活
将 source ~/esp-idf-v5.1/export.sh 添加到 ~/.bashrc 末尾，每次打开终端自动激活。

编译固件

# 进入源码目录
cd xiaozhi-esp32-2.2.6

# 设置目标芯片
idf.py set-target esp32s3

# 配置（可选：修改 Wi-Fi 默认 SSID、配网方式等）
idf.py menuconfig
# 常见配置项：
#   - Wi-Fi 配置 → 默认 SSID/密码
#   - 板型选择 → electronBot
#   - MCP 协议 → 启用

# 编译（首次编译较慢，约 5-10 分钟）
idf.py build

合并固件分区

编译完成后，需要将多个 bin 文件合并为单一固件：

esptool.py --chip esp32s3 merge_bin -o merged-flash.bin \
  --flash_mode dio --flash_size 16MB \
  0x0 build/bootloader/bootloader.bin \
  0x8000 build/partition_table/partition-table.bin \
  0xd000 build/ota_data_initial.bin \
  0x20000 build/xiaozhi.bin \
  0x800000 build/generated_assets.bin

烧录合并固件

esptool.py --chip esp32s3 --port /dev/ttyUSB0 --baud 921600 \
  write_flash 0x0 merged-flash.bin
烧录后验证
烧录完成后，用 minicom -D /dev/ttyUSB0 -b 115200 查看串口输出。
确认看到 Wi-Fi 连接成功和 IP 地址获取日志，然后用 websocat ws://IP:8080/ws 验证 WebSocket 服务。

  1. ESP-IDF v5.1 是编译 xiaozhi-esp32 v2.2.6 的必要工具链
