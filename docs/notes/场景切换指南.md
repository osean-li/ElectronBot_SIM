# 场景切换指南

## 文件结构

```
assets/mjcf/
├── electronbot.xml          # 机器人模型（不可见场景元素）
├── electronbot_scene.xml    # 场景文件（包含场景，include 机器人）
├── go2.xml                  # 参考：另一机器人模型
└── scene.xml                # 参考：go2 的场景文件
```

采用 `go2.xml + scene.xml` 的分离模式：**机器人**与**场景**各自独立，通过 `<include>` 组合。

## 文件职责

### `electronbot.xml` — 机器人

只包含机器人本体，不涉及地面、灯光、天空等场景元素：

- `<mujoco>`, `<compiler>`, `<option>`, `<default>`
- `<asset>` — 材料定义、内联 mesh
- `<worldbody>` — 机器人 body 树（`base_link` → `body` → `head`/`arm`）
- `<actuator>`, `<sensor>`, `<keyframe>`

单独加载时即可运行，不依赖任何场景元素。

### `electronbot_scene.xml` — 场景（可自由修改）

```xml
<mujoco model="electronbot scene">
  <include file="electronbot.xml"/>           <!-- 引入机器人 -->

  <visual>                                     <!-- 视觉效果 -->
    <headlight diffuse="0.8 0.8 0.8" ambient="0.4 0.4 0.4" specular="0.1 0.1 0.1"/>
    <rgba haze="0.15 0.25 0.35 1"/>            <!-- 远景雾效色调 -->
    <global azimuth="-130" elevation="-20" offwidth="960" offheight="720"/>
  </visual>

  <asset>                                      <!-- 场景资源 -->
    <texture type="skybox" builtin="gradient"   <!-- 天空渐变 -->
             rgb1="0.3 0.5 0.7" rgb2="0 0 0"
             width="512" height="3072"/>
    <texture type="2d" name="groundplane"       <!-- 棋盘格地面纹理 -->
             builtin="checker" mark="edge"
             rgb1="0.50 0.60 0.72"             <!-- 亮格颜色 -->
             rgb2="0.06 0.13 0.22"             <!-- 暗格颜色 -->
             markrgb="0.9 0.9 0.9"             <!-- 网格线颜色 -->
             width="300" height="300"/>
    <material name="groundplane"                <!-- 地面材质 -->
             texture="groundplane"
             texuniform="true"
             texrepeat="0.05 0.05"             <!-- 纹理重复次数（越小格子越大） -->
             reflectance="0.12"/>              <!-- 反射率（0-1） -->
  </asset>

  <worldbody>                                  <!-- 场景世界 -->
    <light pos="0 0 1.5" dir="0 0 -1" directional="true"/>
    <geom name="floor" size="0 0 0.05" type="plane" material="groundplane"/>
  </worldbody>
</mujoco>
```

## 常用调整

### 1. 棋盘格地面

| 参数 | 位置 | 效果 |
|------|------|------|
| `rgb1` | `<texture>` | 亮格颜色，调亮可提高对比度 |
| `rgb2` | `<texture>` | 暗格颜色，调暗可提高对比度 |
| `markrgb` | `<texture>` | 网格线颜色 |
| `texrepeat` | `<material>` | 纹理重复次数，**减小**使方格变大（当前 0.05 匹配 ~70m 高模型） |
| `reflectance` | `<material>` | 地面反射率，降低可减弱倒影 |

### 2. 天空 / 远景

| 参数 | 位置 | 效果 |
|------|------|------|
| `rgb1` | skybox `<texture>` | 天空顶部颜色（亮部） |
| `rgb2` | skybox `<texture>` | 天空底部颜色（暗部） |
| `haze` | `<rgba>` | 雾效颜色和强度，alpha=1 最浓 |

### 3. 灯光

| 参数 | 位置 | 效果 |
|------|------|------|
| `diffuse` | `<headlight>` | 漫射光强度 |
| `ambient` | `<headlight>` | 环境光强度（补光） |
| 灯位置 | `<light pos>` | 主光源位置 |

### 4. 相机

在 `demos/.../02_sequence_demo.py` 中：

```python
cam.distance = 280.0      # 相机距离（当前模型 ~70m 高，需 ~280m）
cam.azimuth = 135          # 水平角度（135° = 右前侧）
cam.elevation = 6           # 俯仰角（正 = 俯视，负 = 仰视）
cam.lookat[:] = [0.0, 0.0, 31.0]  # 注视点
```

> **注意**：当前模型 mesh 顶点以 **mm** 被 MuJoCo 解析为 **m**，导致模型整体高约 **70m**，因此相机距离需百米级、棋盘格 `texrepeat` 需很小（`0.05`）。如果未来修复了 mesh 缩放（加 `scale="0.001"`），需同步调整相机距离到 0.5~0.7m、texrepeat 回到 1~2。

### 5. 渲染分辨率

在 `02_sequence_demo.py` 中：

```python
renderer = mujoco.Renderer(model, 720, 960)  # height × width
```

同时需保证 scene.xml 中有对应设置：
```xml
<global offwidth="960" offheight="720"/>
```

## 想换另一个场景

1. 复制 `electronbot_scene.xml` 为新文件
2. 修改 `<worldbody>` 中的地面几何体和灯光
3. 修改 `<asset>` 中的纹理和材质
4. 修改 `<visual>` 中的光效和雾效

无需改动 `electronbot.xml`。

## 踩坑记录 & 解决过程

将原始单文件 `electronbot_full_arm.xml` 拆分为 `electronbot.xml` + `electronbot_scene.xml` 并实现 `scene.xml` 风格的棋盘格场景，经历以下问题：

### 坑 1：XML 解析失败（空文件）

```
Error=XML_ERROR_PARSING_ATTRIBUTE Line number=19: XMLElement name=mesh
```

**原因**：`replace_in_file` 操作将 electronbot_full_arm.xml 写空为 0 字节（文件内有超长内联 mesh 行，编辑器写入异常）。

**解决**：`git checkout -- assets/mjcf/electronbot_full_arm.xml` 恢复，改用 Python 脚本而非直接文本替换处理大文件。

### 坑 2：棋盘格地面显示不出来

场景文件已照搬 `scene.xml` 的检查器纹理设置，但渲染图始终是纯色地面。

**原因**：两层问题叠加——
1. **模型尺寸畸变**：mesh 顶点数值来自 STL 导出（mm），MuJoCo 直接解析为 m，导致机器人高约 70m、宽约 60m。默认相机自动适配了巨物，地面纹理自然不可见。
2. **纹理颜色对比度太低**：`scene.xml` 的 `rgb1="0.2 0.3 0.4"` / `rgb2="0.1 0.2 0.3"` 差异太小，远处雾效一混就成纯色。

**解决**：
- 增大 `rgb1`/`rgb2` 对比度（最终 `0.50 0.60 0.72` vs `0.06 0.13 0.22`）
- 根据模型实际尺寸（70m）设置 `texrepeat="0.05 0.05"`（每格 20m）使棋盘格在远方仍可见

### 坑 3：模型零件显示不完全（缩放爆炸）

尝试用 `scale="0.001 0.001 0.001"` 修正 mesh 尺寸，让机器人变成 ~7cm 高，然后发现：

**问题**：零件散架——底座在 z=-1.5cm、身体在 z=3.5cm、头在 z=15.7cm，关节处肉眼可见断口。

**原因**：原始 body 位置（`base_link z=0.015`、`body z_rel=0.03`、`head z_rel=0.07`）是为 **mm 级 mesh** 设计的。缩放 mesh 但不缩放 body 位置，导致 mesh 质心和 body 相对位置不一致，零件无法首尾相连。

**踩过的错误姿势**：

| 尝试 | 结果 |
|------|------|
| 保持原始 body 位置 + mesh `scale=0.001` | 零件分散，关节外露 |
| 根据 mesh 边界精确计算 body 位置 | 零件完美对齐但 "太整齐"，失去原始设计的间隙/重叠；计算复杂易出错 |
| 给 body 位置也乘以 0.001 | 机器人变成 0.1mm 厚的"贴纸" |

**最终决定**：**放弃缩放**，接受 ~70m 的巨人模型。相机拉远（280m）即可正常框住，棋盘格 texrepeat 调小适配大尺寸。

### 坑 4：机器人悬空，不在地面上

底座底部在 z=-47m，地面在 z=0，悬空 47m。

**原因**：`base_link pos z=0.015` 仅定义了 body 原点位置，而 mesh 底部在 -47m。

**解决**：`base_link pos z = -mesh_min_z = -(-47) = 47.0`，同步更新 camera `lookat z = 31.0`（机器人中心高度）。

### 坑 5：`MUJOCO_GL=egl` 总是忘加

运行 demo 报 `GLFWError: DISPLAY environment variable is missing`（无桌面环境）。

**解决**：在脚本开头 `os.environ.setdefault("MUJOCO_GL", "egl")`，自动选择 EGL 后端，不再需要手动加环境变量前缀。

### 坑 6：渲染分辨率超过默认帧缓存

把 `Renderer` 从 `480×640` 提升到 `720×960` 时报错。

**解决**：在 scene.xml 的 `<visual>` 中添加 `<global offwidth="960" offheight="720"/>`。

### 经验总结

| 问题 | 根因 | 规范做法 |
|------|------|---------|
| 棋盘格不可见 | 模型尺寸畸变 + 纹理对比度低 | 先确认模型实际尺寸再调相机/纹理 |
| 零件断开 | 缩放 mesh 后 body 位置未同步 | **不改 mesh，只调相机** |
| 机器人悬空 | body 与 mesh 底部的偏移 | 设 `base_link z = -mesh_min_z` |
| 渲染失败 | 缺 GLFW/EGL | 脚本自动注入 `MUJOCO_GL=egl` |
| 分辨率超限 | 未设 offscreen framebuffer | scene.xml 加 `offwidth/offheight` |
