# dev-note: 修复 ElectronBot 渲染与举手动作 (v2)

## 背景

用户报告: 
- 启动 `python -m electronbot_sim.mcp_server` 后, 机器人**默认状态和举手状态显示都错**
- 期望: 模型外观匹配 FreeCAD 渲染的 `assets/cad/ElectronBot.step` (圆头+椭球身体+两侧"耳朵"手臂+双层底座)
- hand_action raise (action=1) 后**双手应举到头部两侧上方**

## 根本原因分析

### 1. STL mesh 的几何不匹配目标外观

原 5 个 STL mesh (base_link/body/head/left_arm/right_arm) 是从 FreeCAD 按"机械零件组"导出的, 包含**内部结构** (齿轮、舵机臂、PCB), 不是目标图的"光滑封闭"外观。

**修复**: **完全弃用 STL mesh**, 改用 MuJoCo 原生几何体:
- `ellipsoid` (椭球) 用于身体、头、手臂耳、手
- `box` (长方体) 用于手臂
- `cylinder` (圆柱) 用于底座
- `sphere` (球) 用于眼睛、手、装饰
- 配合 `material` 渲染 (白色+金色+深色眼睛)

### 2. position actuator 数值稳定性问题

`apply_joint_targets_deg` 之前只写 `data.ctrl[aid] = target_rad[i]`, 让 mujoco position actuator 的 PID 慢慢逼近目标。**当 `dt=0.002, mass=0.05kg, kp=200, kv=20` 时**, 由于:
- `arm body inertia about pitch axis ≈ 0` (圆柱沿 Y 轴 + 手在 Y 轴), 实际只有 1e-5 kg·m² 量级
- `qfrc = kp * (ctrl - qpos) = 200 * 1.57 = 314 N·m`
- `a = qfrc / I = 314 / 1e-5 = 3e7 rad/s²` (**单步角速度变化 60 rad!**)
- **dt=0.002 步长无法稳定, 系统震荡到 ~70° 卡住 (稳态 qpos 71°)**

**修复**: `env.py: apply_joint_targets_deg` **同时**写 `data.ctrl` (兼容) **和直接 set `data.qpos`** (绕过 PID). 这样舵机一步到位, 无需等待 PID 收敛.

### 3. 手臂 mesh 几何 + axis 反向问题

之前用 STL mesh 时手是 box 形. 改用 box + sphere 后:
- arm geom 用 `box` (沿 X 方向 50mm) 给 arm body **足够的 X 轴 inertia**, 解决 PID 数值问题
- 之前用 cylinder (沿 Y 轴), inertia 沿 Y 是 0, 导致 PID 数值问题

另外, `right_pitch_joint axis="0 -1 0"` 让 RP=+90 (舵机 0) 让右手向上 (从 +x 转到 +z). 配合 `right_hand 起始位置 (ARM_LENGTH, -ARM_LENGTH/2, 0)` 让 hand 在 +X 端部.

左手对称: `left_pitch_joint axis="0 1 0"`, `left_hand` 起始位置 `(-ARM_LENGTH, -ARM_LENGTH/2, 0)`.

### `src/electronbot_sim/env.py` (关键 patch)

`apply_joint_targets_deg` 直接 set qpos (绕过 PID):

```python
def apply_joint_targets_deg(self, joint_angles_deg):
    angles = np.clip(angles, self._joint_min, self._joint_max)
    target_rad = np.radians(angles)
    # 同时写 ctrl (兼容) 和 qpos (直接到位, 绕过 PID 数值稳定性)
    for i, aid in enumerate(self._act_ids_sorted):
        self.data.ctrl[aid] = target_rad[i]
    for i, addr in enumerate(self._qpos_addr):
        self.data.qpos[addr] = target_rad[i]
    for addr in self._qvel_addr:
        self.data.qvel[addr] = 0
    self._mujoco.mj_forward(self.model, self.data)
    self._last_commanded = angles.copy()
```
