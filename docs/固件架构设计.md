## 4. 固件对齐约束清单 (核心)

### 4.1 线性插值 (对齐 movements.cc:87)

```
固件 MoveServos 实现:
    increment_[i] = (target[i] - pos[i]) / (time / 10.0)
    每 10ms 步进等量增加 → 纯线性等分

仿真实现 (mcp_bridge.py:_move_all_servos):
    steps = max(1, time_ms // 10)
    for step in 1..steps:
        t = step / steps  # 线性进度 0→1
        interp = current + (target - current) * t

⚠️ 禁用 EaseOutCubic 或任何缓动函数
```

### 4.2 安全角度裁剪 (对齐 ClampServoTarget)

| 索引 | 舵机 | 下限(°) | 上限(°) | 中心(°) |
|:---:|------|:---:|:---:|:---:|
| 0 | right_pitch | 0 | 180 | 180 |
| 1 | right_roll | 100 | 180 | 140 |
| 2 | left_pitch | 0 | 180 | 0 |
| 3 | left_roll | 0 | 80 | 40 |
| 4 | body | 30 | 150 | 90 |
| 5 | head | 75 | 105 | 90 |

### 4.3 舵机→关节映射比 (单一数据源)

| 关节 | 映射比 | 方向 | 转换公式 |
|------|:---:|:---:|------|
| HEAD | 2.0 | +1 | joint = (servo-90) × 2.0 |
| BODY | 1.5 | +1 | joint = (servo-90) × 1.5 |
| RP | 1.0 | -1 | joint = (servo-180) × -1.0 |
| LP | 1.0 | +1 | joint = (servo-0) × 1.0 |
| RR | 1.125 | -1 | joint = (servo-140) × -1.125 |
| LR | 1.125 | +1 | joint = (servo-40) × 1.125 |

**所有模块必须从 `env.py` 导入这些常量, 禁止重复定义。**

### 4.4 控制频率

| 参数 | 值 | 对齐 |
|------|:---:|------|
| 控制频率 | 50 Hz | 真机 50Hz PWM (20ms 周期) |
| 控制步长 dt | 0.02 s | 1/50Hz |
| 仿真子步数 | 10 | 0.02/0.002=10 |
| 子步步长 | 0.002 s | MuJoCo 底层积分步长 |
| 插值步进 | 10 ms | 对齐固件 time/10.0 |
| 振荡器采样 | 50 ms | 对齐 vTaskDelay(5) @ 100Hz |

### 4.5 hand_action times 限制

```
固件: times = 2 * max(3, min(100, steps))
仿真: 完全复现 (mcp_bridge.py:_hand_action)
```


## 7. Sim2Real 三种部署路径

### 7.1 路径对比

| 路径 | 通信方式 | 延迟 | 当前可用 | 适用场景 |
|------|----------|:---:|:---:|------|
| **A: 云端 API 透传** | HTTPS→云端→MQTT/WS→ESP32 | 200-500ms | ✅ | 预设动作/VLA |
| **B: ONNX 推理+降级** | ONNX→降级→云端 API | 200-500ms | ✅ | RL/IL 策略 |
| **C: WebSocket 直连** | ws://IP:8080/ws | <10ms | ❌ 需 OTA | 实时闭环控制 |

### 7.2 关键约束

```
⚠️ release v2.2.6 真机 ESP32 是 WebSocket 客户端 (连接云端后台)
   不是 Server, 真机没有 ws://IP:8080/ws 端点
   所有真机通信必须通过小智云端 API 透传

⚠️ 云端 API 延迟 200-500ms RTT (对应 10-25 仿真步)
   有效闭环延迟 400-1000ms (对应 20-50 仿真步)
   → PPO@50Hz 实时策略不可通过云端部署
   → 仅支持预设动作调用 (LLM/VLA 场景)
   → 实时控制必须 OTA 升级后使用路径 C (<10ms)
```

### 7.3 能力降级策略

| 仿真命令 | 降级目标 | 精度损失 |
|---------|---------|:---:|
| `servo_move` | `hand_action` (按角度方向) | 高 |
| `servo_sequences` | `hand_action`+`body_turn`+`head_move` | 中 |
| `home` | `stop` (功能等价) | 无 |
| `osc` (振荡器) | 多次 `hand_action` | 高 |

## 9. 风险与缓解措施

| 风险 | 影响 | 缓解措施 | 状态 |
|------|------|---------|:---:|
| 角度单位混用 | 57.3 倍偏差 | env.apply_joint_targets_deg() 统一入口 | ✅ 已解决 |
| 映射比多源定义 | 数据不一致 | env.py 单一数据源, 其他模块导入 | ✅ 已解决 |
